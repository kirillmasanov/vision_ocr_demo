Нужно разработать приложение для демонстрационного сценария сервиса Yandex Vision OCR.
Описание сервиса Yandex Vision OCR находится на странице https://github.com/yandex-cloud/docs/blob/master/ru/vision/concepts/ocr/index.md  изучи его.
Нужно сделать веб-интерфейс, который демонстрирует всю функциональность сервиса. 
Бэкэнд приложения написан на FastApi. Управление зависимостями происходит с помощью uv. Все пути в эндпоинтах - относительные. Имеется в том числе эндпоинт /api/health.
Веб интерфейс я вижу таким что: на экране находится две кнопки - "Модели для распознавания текста" и "Модели для распознавания шаблонных документов".
При выборе например, "Модели для распознавания текста" появляется возможность выбрать из выпадающего списка одну из моделей соответствующую "Модели для распознавания текста", согласно приложенной выше документации (например, page, page-column-sort, handwritten и другие).
При нажатии на кнопку "Модели для распознавания шаблонных документов" соответственно будет выпадающий список моделей passport, driver-license-front и так далее. 
После выбора модели будет выведено ее описание согласно документации, например, если выбрана модель page, то описание будет - "подойдет для изображений с любым количеством строк текста, сверстанного в одну колонку".
После выбора модели ниже будут подгружены миниатюры изображений (тестовые примеры для дальнейшего распознования). Все эти тестовые изображения находятся в каталоге проекта в папке static в соответсвующих папках для каждой модели. Название папки - есть название модели. Кол-во миниатюр соответствует количеству файлов для распознования в соответствующих каталогах. 
Так же должна быть кнопка для возможности загрузки своего файла для дальнейшего распознования.
Предварительно под миниатюры сделай заглушки, я потом в соответствующие папки загружу примеры изображений.
Аутентификация через API ключ из файла env. (YANDEX_API_KEY, YANDEX_FOLDER_ID).
Так должна иметься кнопка - Распознать. После нажатия на эту кнопку будет выведен результат. А так дополнительно сырой результат json по катом.
